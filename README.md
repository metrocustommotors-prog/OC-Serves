# OC Serves — Website

Marketing website for OC Serves, a professional process serving company
based in Newport Beach, Orange County, CA.

## What this is

A static website — plain HTML, CSS, and JavaScript. No build step, no
dependencies, no framework. It will run on any static host and is fast
and easy for search engines to crawl.

## Deploying to GitHub + Bolt

1. Create a new repository on GitHub and push every file in this folder
   to it (keep the folder structure intact).
2. In Bolt, import the GitHub repository.
3. Because this is a plain static site, there is no build command. If
   Bolt asks:
   - Build command: leave blank (or `echo "static"`)
   - Publish/output directory: `.` (the repository root)
4. Deploy. The `netlify.toml` file handles the 404 page and security
   headers automatically on Netlify-compatible hosts.

## Important: things to do after launch

### 1. The contact form
The form on `contact.html` currently opens the visitor's email client
pre-filled to `docs@ocserves.com`. This works everywhere with zero
setup, but it depends on the visitor having an email client.

For a more reliable form, connect a form service (e.g. Formspree or
Netlify Forms). With Netlify Forms you would add `data-netlify="true"`
to the `<form>` tag and a hidden `form-name` field; the form then
submits to your dashboard. Ask your developer to wire this up.

### 2. Google Business Profile
The single biggest factor for ranking in local "near me" searches is a
verified Google Business Profile, which is separate from this website.
Create/claim it at google.com/business with the exact same name,
address, and phone number used on this site.

### 3. Submit the sitemap
In Google Search Console, add the property `www.ocserves.com` and
submit `https://www.ocserves.com/sitemap.xml`.

### 4. Verify the registered-agent details
The Registered Agents page lists Experian (475 Anton Blvd, Costa Mesa)
and Deutsche Bank (1761 E. St. Andrew Place, Santa Ana) as locations
served. Registered-agent addresses change — re-confirm periodically.

## Adding more city pages later

City pages are generated by `generate_cities.py`. To add a city:

1. Open `generate_cities.py` and add a new entry to the `CITIES` list,
   following the format of the existing entries. Each city needs
   genuinely distinct content (courts, local context) — this is what
   keeps the pages from being treated as low-quality "doorway pages".
2. Run: `python3 generate_cities.py`
3. Add a link to the new page in `cities/index.html`.
4. Add the new URL to `sitemap.xml`.

## A note on SEO strategy

This site uses ~25 substantive city pages rather than hundreds of thin
ones. Search engines penalize large sets of near-identical pages. The
real ranking strength here is: a strong Google Business Profile for
local Orange County searches, plus genuinely useful service and city
pages, plus the registered-agent and HOA pages which target specific,
lower-competition searches.
